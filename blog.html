1: What is the differences between Local Storage and Session Storage?

Local storage and session storage are found at application tab in Developer tools. Though they both work in the same process but there are some differences which make them different. Take a look:

Data stored in Local storage has no expiration date. If once data is saved in local storage, then it remains until user remove the data. That means, browser doesn't take any step to remove the data.

On the other hand, Session storage's data erased after closing the browser or tab! So, it has limit and expiration date.

2: What is the differences between Global Scope and Block Scope?

Scope means the region of a program where a variable can be accessible. There are two types of scope in javascript.

Global Scope
Global means the entire program region. If a variable or function declared in global scope, it can be accessible in anywhere of the program.

Local Scope
Local Scope has limited access region. If we define any variable inside the curly brace, { }, then those variable are local to that region. Though we can access global variable inside the local scope, local scope's variable isn't accessible outside the local scope.

3: How does Javascript Event-Loop work?

Javascript's event loop is responsible for executing the code and other stuffs of events. When we talk about event loop, there comes up two topics: Stack and Queue.

We know javascript is a single threaded language. It's a asynchronous language. When a javascript code executes, the event are stored in Stack which is 'First-In Last-Out'. At the same time, some events like setInterval or fetch are kept in Queue which is 'First-In First-Out'. And after processing the synchronous tasks, queue's tasks are joined to executed.

4: How many ways are there to get 'undefined'?

There are many ways responsible to produce 'undefined'. Some of them are listed below:

  • Variable that is not initialized will give undefined
  • A function that has no return statement
  • A parameter that is not passed will be undefined
  • Only return keyword returns undefined value
  • A object property that doesn't exists weill produce undefined
  • If array index is out of the range of array, it certainly provide undefined
